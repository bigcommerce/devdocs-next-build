store/customer/payment/instrument/default/updated

This webhook fires when a customer default payment instrument is updated.

Example callback:

{
  "created_at": 1561482670,
  "store_id": "1025646",
  "producer": "stores/{store_hash}",
  "scope": "store/customer/payment/instrument/default/updated",
  "hash": "352e4afc6dd3fc85ea26bfdf3f91852604d57528",
  "data": {
    "type": "customer",
    "id": 32
  }
}

Callback schema:

object
created_at
integer

Hook creation date as a Unix timestamp.

2 validations
store_id
string

A numerical identifier that is unique to each store.

2 validations
producer
string

Will always follow the pattern stores/store_hash. This is the store that created the webhook.

2 validations
scope
string

The event registered when the webhook was created.

2 validations
hash
string

The payload data JSON encoded then passed through SH1 encryption.

2 validations
data
object

A lightweight description of the event that triggered the webhook. Will vary depending on the event registered.
